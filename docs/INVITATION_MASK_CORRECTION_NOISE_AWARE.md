## Interpretation

The feedback-noise sweep should be read as a reliability boundary. Mask correction reduces missed opportunities by trusting the aggregate current feedback count, but larger count noise can turn that trust into extra failed invitations. The clipped formal result resolves the high-noise side of this boundary by limiting per-slot target-count changes instead of changing the IRS candidate-generation stage.

The broad pilot showed that positive deadband values (`z>0`) suppress useful corrections and are not worth formalizing. The formal clipped variant therefore uses `z=0` and only caps the per-slot target-count delta. `mc=1 clip=2` is not the reliable-feedback main method because it slightly worsens gap at noise std `0`, `0.02`, and `0.05`; its value is high-noise robustness. At noise std `0.1`, it improves direct `mc=1` from gap/failed/missed `0.856/5.264/0.734` to `0.818/3.275/0.599`.
